# FREEZE! Mouse Freeze Challenge 기획서

## 1. 프로젝트 개요

`FREEZE!`는 마우스 또는 터치 포인터를 얼마나 정밀하게 제어할 수 있는지 테스트하는 웹 미니게임입니다. 사용자는 제한된 시간 안에서 포인터를 움직이다가 정확한 순간 멈추거나, 신호 전환에 맞춰 움직임과 정지를 반복합니다.

이 프로젝트는 단순한 반응속도 게임보다 한 단계 더 좁은 감각인 **정지 타이밍**, **미세 움직임 제어**, **주의 전환**을 게임화하는 것을 목표로 했습니다.

## 2. 왜 만들었는가

브라우저 미니게임은 짧은 시간 안에 제품 감각, 프론트엔드 구현력, 배포 역량을 동시에 보여주기 좋은 형식입니다. `FREEZE!`는 다음 목적을 위해 제작했습니다.

- 아이디어를 빠르게 실행 가능한 웹 제품으로 전환하는 능력 검증
- 마우스 움직임이라는 단순 입력을 게임 규칙으로 재구성
- Vite, TypeScript, Tailwind 기반의 가벼운 배포형 프로젝트 구축
- GitHub 공개 repo와 Vercel production 배포까지 이어지는 실전 워크플로우 확보

특히 `마우스를 움직인다`가 아니라 `정확히 멈춘다`를 핵심 행동으로 삼아, 기존 클릭/타이밍 게임과 다른 차별점을 만들었습니다.

## 3. 무엇을 만들었는가

### 핵심 모드

- `Target Freeze`
  - 3초에서 15초 사이의 랜덤 목표 시간이 주어집니다.
  - 사용자는 포인터를 움직이다가 목표 시간에 맞춰 멈춥니다.
  - 목표 시간 직전에는 타이머를 가려 단순 눈치 게임이 아니라 감각 기반 플레이가 되도록 설계했습니다.

- `Red Light`
  - 초록 신호에서는 포인터를 움직여 점수를 채웁니다.
  - 빨간 신호에서는 즉시 정지해야 합니다.
  - 신호 전환 직후에는 짧은 완충 시간을 두어 불합리한 오탐을 줄였습니다.

### 보조 기능

- 최근 5개 플레이 기록 저장
- Target Freeze 최고 오차 표시
- 브라우저 `localStorage` 기반 기록 유지
- 사용자 interaction 이후 활성화되는 Web Audio 효과음
- 데스크톱과 모바일 포인터 이벤트 대응

## 4. 어떻게 만들었는가

### 기술 스택

- `Vite`: 빠른 개발 서버와 정적 배포에 적합해 선택했습니다.
- `TypeScript`: 게임 상태, 결과 타입, 기록 구조를 명확히 관리하기 위해 사용했습니다.
- `Tailwind CSS`: 빠른 UI 조립과 반응형 레이아웃 구현을 위해 사용했습니다.
- `Lucide`: 버튼과 상태 표현에 일관된 아이콘 시스템을 적용하기 위해 사용했습니다.
- `Vercel`: 정적 프론트엔드 앱을 빠르게 production 배포하기 위해 사용했습니다.

### 구조 설계

게임 구현은 단일 HTML 스크립트에서 출발했지만, 배포 가능한 프로젝트로 만들기 위해 관심사를 분리했습니다.

- `src/game/simulation.ts`: 목표 시간, 점수 계산, 결과 생성 등 순수 게임 규칙
- `src/game/input.ts`: pointer event 기반 입력 감지와 움직임 판정
- `src/game/storage.ts`: localStorage 기록 저장과 최고 기록 계산
- `src/game/audio.ts`: Web Audio 기반 효과음
- `src/main.ts`: 화면 렌더링, 상태 전환, 이벤트 연결

이 구조는 렌더링과 게임 규칙을 분리해 추후 난이도, 랭킹, 신규 모드 추가가 쉬운 형태입니다.

## 5. 디자인 방향

디자인은 `정밀한 조작 실험실` 감성을 기준으로 잡았습니다. 어두운 배경, 디지털 타이머, 얇은 라인, 신호 색상을 사용해 게임의 핵심인 집중과 제어를 시각적으로 강화했습니다.

모바일에서는 설명보다 플레이 영역을 먼저 보이도록 배치했습니다. 미니게임은 첫 화면에서 바로 시작할 수 있어야 하므로, 조작 영역을 우선하고 기록과 모드 선택은 아래로 배치했습니다.

## 6. 주요 의사결정

- Phaser를 사용하지 않고 Vite + TypeScript로 구현했습니다.
  - 이 게임은 복잡한 물리, 타일맵, 스프라이트 애니메이션보다 DOM UI와 pointer event가 핵심이기 때문입니다.
- 서버 랭킹은 v1 범위에서 제외했습니다.
  - 빠른 공유와 배포가 우선이므로, 브라우저 로컬 기록만 유지했습니다.
- 타이머를 목표 시간 직전에 가렸습니다.
  - 사용자가 화면 숫자를 그대로 따라가는 것을 줄이고, 감각 기반 게임성을 강화하기 위한 결정입니다.
- 모바일 터치를 고려해 `pointer event`를 사용했습니다.
  - 마우스와 터치를 하나의 입력 모델로 다룰 수 있어 유지보수가 쉽습니다.

## 7. 검증 기준

완성 기준은 단순 실행이 아니라, 실제 공유 가능한 production 품질로 잡았습니다.

- `npm run typecheck` 통과
- `npm run build` 통과
- 데스크톱/모바일 화면에서 레이아웃 깨짐 없음
- Target Freeze 시작, 정지, 결과, 기록 저장 플로우 정상
- Red Light 시작, 진행, 점수 표시 플로우 정상
- Vercel production 배포 후 접속 및 기본 플레이 확인
- 콘솔 오류 없음
